# Source Optimization Workflow

## Purpose
Improve the trusted source set so the job-search system is sharper, safer, easier to use, and less token-heavy.

- `source-files/Wealth Dynamics Report - Matt Dimock - 2025.pdf`, internal operating-style input

## Audit Goals
- Remove conflicting positioning.
- Reduce duplicate or bloated language.
- Strengthen role-lane clarity.
- Improve claim safety and metric traceability.
- Make the files easier for Codex to load selectively.
- Improve recruiter-facing clarity and conversion.
- Preserve facts, context, and approval gates.

## Expert Lenses
Use these lenses during audit:
- executive recruiter
- ATS resume architect
- hiring manager psychologist
- GTM / RevOps strategist
- ecommerce growth strategist
- lifecycle / CRM strategist
- compensation strategist
- executive communications strategist
- QA and fact-check reviewer
- token-budgeting reviewer

## Workflow
1. Inventory current source files and modified dates.
2. Identify contradictions, duplicated material, stale phrasing, and unsafe claims.
3. Propose edits by file, with purpose and risk.
4. Separate external-facing improvements from internal operating notes.
5. Estimate token-efficiency improvements, such as summary tables or index sections.
6. Ask for approval before editing source-of-truth files.
7. After edits, run claim-safety and source-map checks.

## Output
- Source audit findings
- Recommended edits
- Risk level by edit
- Files to update
- Approval request before mutation
- Verification checklist
