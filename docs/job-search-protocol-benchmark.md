# Job Search Protocol Benchmark

Use this file to benchmark whether protocol changes improve application quality, efficiency, and results over time.

## How To Use
- Add one row when a protocol change is materially tested on a live role or a meaningful dry run.
- Keep the scoring consistent. If evidence is weak, leave notes instead of inventing precision.
- Scores use a 1 to 5 scale unless otherwise noted.

## Dimensions
- `ATS fit`: Match Score direction, JD term coverage, and whether key terms appear in recruiter-readable prose.
- `Asset quality`: claim safety, mandate fit, readability, and naming discipline.
- `Page discipline`: whether the resume stays at two pages while using the space well.
- `Workflow efficiency`: time and effort relative to result quality, including avoidable rework.
- `Submission readiness`: whether the package reached filled, ready-to-submit state cleanly.
- `Outcome signal`: later interview or recruiter response evidence, if known.

## Composite Scoring
- Convert each 1 to 5 score into a 20-point band.
- `Protocol health score = (ATS fit + Asset quality + Page discipline + Workflow efficiency + Submission readiness) / 25 * 100`
- Track trendline by comparing the protocol health score across roles and noting which changes drove the shift.
- Do not over-interpret one role. Look for movement across at least 3 meaningful runs before locking in a major workflow rule.
