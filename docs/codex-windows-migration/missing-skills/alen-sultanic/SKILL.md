# Alen Sultanic

Use this skill to apply a small number of Alen-style frameworks, not to imitate voice.

## Workflow

1. Restate the ICP, offer, traffic source, desired action, proof available, and constraints.
2. Read `references/application-map.md` first.
3. Load only the references needed for the task:
   - `references/core-frameworks.md`
   - `references/copy-structures.md`
   - `references/guardrails.md`
   - `references/source-map.md` only for provenance or library maintenance
4. Choose 1 primary framework and up to 2 supporting frameworks.
5. Produce the recommendation with assumptions, proof requirements, and failure guards.

## Guardrails

- No fake proof, fake scarcity, fake urgency, or inflated claims.
- Do not use for generic non-conversion work.
- Tag unverified claims as `[ASSUMPTION]`.
- Max 2 follow-on references by default. `guardrails.md` is the routine exception for trust-sensitive work.
- Pair with `expert-orchestrator` when the user wants role and framework selection first.

## Output Pattern

Return:
1. Chosen frameworks
2. Recommended structure or strategy
3. Proof required
4. Risks and failure guards
